# 更新日志

本文件记录本项目的重要变更，与英文 [CHANGELOG.md](CHANGELOG.md) 语义对齐。

## [0.2.1] - 2026-04-02

### 重点更新

* 修复 `predicate` 层对多个 `$ne` 的合并缺陷，将其正确归一化为单个 `$nin`。
* 修复 `predicate` 层对多个 `$nin` 的合并缺陷，将其正确归一化为单个 `$nin`（值并集），并保持按 `AND` 语义收敛。

### 变更

#### Predicate 层：修正 `$ne`/`$nin` 合并语义

* 默认 `predicate` 改写面现在包含 `ne.ne` 与 `nin.nin` 能力规则。
* 调整默认能力顺序，确保合并规则按期望的执行序列生效。

### 测试

* 新增并扩展单元/语义等价测试：覆盖 `$ne + $ne`、`$nin + $nin`、以及混合 `$ne`/`$nin` 场景。

## [0.2.0] - 2026-04-01

### 重点更新

* 围绕“保守正确性”重新收敛规范化模型。
* 收紧 `predicate`，仅保留更小且显式支持的改写面。
* 重构高层规范化（`scope` / logical），聚焦继承约束传播、保守剪枝与覆盖消除。
* 移除未完成的 experimental 层。
* 扩展契约、单元与语义测试，覆盖高风险 MongoDB 查询语义场景。

### 变更

#### Predicate 层：收敛为显式支持的保守改写面

`predicate` 现定位为保守规范化层：只在小范围、已测试能力内改写；超出能力边界时优先保留原谓词，不做激进假设。

当前已验证的谓词改写能力仅包括：

* `eq.eq`
* `eq.ne`
* `eq.in`
* `eq.range`
* `range.range`

对高风险或语义敏感组合保持保守，不做猜测性改写。包括但不限于：`null` 与字段缺失语义、数组敏感匹配、`$exists`、`$nin`、整对象与点路径混用、以及 opaque/未支持算子与已支持谓词混合。

#### Scope / logical 重构

高层规范化已重构为：

* 继承约束传播
* 保守分支剪枝
* 覆盖消除
* `$or` 公共谓词能力默认降为仅观测

目标更明确：只有在正确性足够清晰时才改写。

#### 移除 experimental 层

未完成的 experimental 层已移除。

#### 简化公开接口面

移除不受支持或具有误导性的预览选项，包括如 `allowNorPropagation` 这类不支持的 safety 选项。

#### 不可满足选择器 canonical 形式更新

不可满足选择器的 canonical 表示已更新。若你的测试或快照依赖“不可能查询”的规范化输出，请同步更新预期。

### 测试

本版本显著加强了 predicate 相关验证。

新增/扩展覆盖：

* `$exists: false` 组合场景
* `$nin` 与已支持谓词改写混用
* `null` 与字段缺失语义
* 可比较与不可比较的 range 边界
* 整对象与点路径交互
* `$regex` / `$elemMatch` 等 opaque 算子混用
* observe/meta 契约一致性
* capability 白名单约束
* 幂等性与语义等价

发布验证包含更完整的 unit/contract 覆盖，以及 Mongo 实例回放的高风险谓词语义验证。

### 破坏性/行为变化

本版本可能改变以下预览层级的规范化输出：

* `predicate`
* `scope`

本版本中，`scope` 内与 logical 处理相关的行为也更偏保守。

这些变化是有意的，体现了更严格、更保守的规范化契约。

`shape` 仍是最稳定、生产最安全的基线，整体定位不变。

### 给使用者的说明

现在应将 `predicate` 理解为：

* 在显式测试覆盖的改写面内可安全使用
* 在改写面外保持保守
* 不是完整的 MongoDB 语义优化器

如需最高可预测性，继续使用 `shape`。若你使用 `predicate` 或更高预览层，升级时请复查快照与断言预期。

## [0.1.0] - 2026-03-30

`mongo-query-normalizer` 的首次公开发布。

### 新增

* 发布初始的分层规范化 API：

  * `shape`
  * `predicate`
  * `scope`
  * `experimental`
* 通过 `meta` 提供可观测的规范化元数据，包括变更标记、规则轨迹、告警、哈希与可选统计。
* 在非生产环境下，对非 `shape` level 输出控制台提示，以降低开发阶段误用的概率。

### 说明与边界

* 在 **v0.1.0** 中，**`shape` 是唯一建议用于一般生产环境的 level**。
* 默认行为仍为 `level: "shape"`。
* **`predicate`、`scope`、`experimental` 作为预览 / 实验能力发布**，适用于：

  * 离线分析
  * 回放测试
  * 语义验证
  * 定向实验
* 在 **v0.1.0** 中，**更高 level 不建议作为一般生产流量的默认选择**。
* 应将本库理解为**可观测、以 shape 为先的规范化层**，默认路径**偏保守、面向一般生产**，而非 MongoDB 查询规划器或优化器。

### 备注

* README 已加强表述，使 **v0.1.0 稳定性边界**更加明确。
* 非 `shape` level 会在每次调用的 **`meta.warnings`** 中写入边界提示。
* 非 `shape` level 在**非生产环境**下还会按 level **至多触发一次** `console.warn`。
* **v0.1.0** 的兼容性承诺**有意不包含**除 `shape` 外其他 level 的「生产就绪」保证。
